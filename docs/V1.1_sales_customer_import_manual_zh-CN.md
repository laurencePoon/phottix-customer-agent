# Phottix Customer Agent V1.1 业务员客户资料导入说明

适用网站：`https://agent.phottix.cn`

## 1. 谁可以导入客户

- `user` 业务员：可以新增和导入客户，不能导出客户。
- `admin` 管理员：可以导入、导出和管理客户。
- `product_manager` 产品经理：当前不能导入客户资料，只能管理产品相关内容。

客户资料应在正式网站导入，不需要操作 Git、Aliyun、SQLite 或 `.env`。

## 2. 客户 CSV / Excel 的格式

支持的文件格式：`.csv`、`.xlsx`、`.xls`。

第一行必须是英文栏位名称，第二行开始填写客户。每一行代表一个客户公司。

推荐栏位：

| 栏位 | 是否建议填写 | 填写规则 |
|---|---:|---|
| `Company Name` | 必填 | 公司正式名称；不要重复或随意缩写 |
| `Website` | 强烈建议 | 完整网址，例如 `https://example.com` |
| `Contact Name` | 建议 | 主要联系人姓名 |
| `Contact Email` | 建议 | 主要联系人邮箱，确认格式正确 |
| `Country` | 建议 | 国家或地区，例如 `Hong Kong` |
| `City` | 可选 | 城市，例如 `Hong Kong` |
| `Industry` | 建议 | 客户所属行业，例如 `Photography Retail` |
| `Customer Type` | 建议 | 只能填写 `prospect` 或 `existing` |
| `Buying Role` | 建议 | 只能填写系统支持的购买角色 |
| `Customer Score` | 可选 | 只能填写 `1` 至 `100` 的数字 |
| `Group` | 可选 | 客户组名称；正式上传时以网页的 Target Group 选择为准 |
| `Follow-up Status` | 可选 | `open`、`pending`、`completed`、`cancelled` 或 `deferred` |
| `Next Follow-up Date` | 可选 | 建议使用 `YYYY-MM-DD` |
| `Last Contact Date` | 可选 | 建议使用 `YYYY-MM-DD` |
| `Notes` | 可选 | 客户备注，不要放内部密码或敏感资料 |
| `Email Purpose` | 可选 | 例如 `First Touch`、`Quotation Follow-up` |
| `Company Type` | 可选 | 例如 `Retailer`、`Distributor`、`Studio` |
| `Main Products` | 可选 | 客户主要销售或使用的产品 |
| `Instagram` | 可选 | Instagram 网址或账号 |
| `Facebook` | 可选 | Facebook 网址或账号 |
| `Force_Update` | 不要默认填写 | 只有确认要更新重复客户时才填写 `YES` |

至少应填写 `Company Name`、`Website`、`Contact Email` 三项中的一项；最好三项都填写。

## 3. Buying Role / 买家分类的正确值

这是产品推荐的关键栏位。必须先判断客户属于 A、B、C 或 D，再匹配相同目标的产品 Priority；`All` 产品会作为该客户的通用推荐一起参与排序。

| 代码 | 客户分类 | 判断重点 |
|---|---|---|
| `A` | 批发商 / Wholesaler | 批量采购、分销、代理、经销网络、B2B 或进口商 |
| `B` | 实体零售商 / Physical Reseller | 有实体店、门店、展厅、展示空间或线下销售 |
| `C` | 网店零售商 / Online Reseller | 网店、电商、Shopify、购物车、线上结账或线上销售 |
| `D` | 工作室 / Studio or End User | 摄影棚、制作团队、摄影师、摄像师、内容创作者或最终使用者 |
| `Unknown` | 未分类 | 资料不足或同时符合多个类别，需人工确认 |

CSV / Excel 的 `Buying Role` 栏只能填写 `A`、`B`、`C`、`D` 或 `Unknown`。系统也兼容旧资料中的 `Wholesaler`、`Reseller - Physical`、`Reseller - Online` 和 `End User - Studio`，导入后会自动转换为 A/B/C/D。

如果网站资料不足，先填写 `Unknown`，不要猜测。打开客户资料后，可在 `Buyer Classification` 下拉菜单人工选择正确类别；人工选择后，重新分析不会覆盖该选择。

产品推荐规则：

- A 客户：匹配产品的 `A-1`、`A-2`、`A-3`，并同时纳入 `All-1`、`All-2`、`All-3`。
- B 客户：匹配 `B-1`、`B-2`、`B-3`，并同时纳入 `All`。
- C 客户：匹配 `C-1`、`C-2`、`C-3`，并同时纳入 `All`。
- D 客户：匹配 `D-1`、`D-2`、`D-3`，并同时纳入 `All`。
- `A/B/C/D/All` 没有总体主推荐；每个目标的 Priority 独立生效。

## 4. Customer Type 和 Customer Score

`Customer Type` 只能填写：

- `prospect`：潜在客户
- `existing`：现有客户

`Customer Score` 是业务员的参考分数，范围为 `1` 至 `100`。它不会直接改变产品推荐结果，不确定时可以留空。

## 5. 在 agent.phottix.cn 上传

1. 登录 `https://agent.phottix.cn`。
2. 进入「客户池 / Customer Pool」。
3. 在「目标组别 / Target Group」选择要放入的客户组；如果不选择，客户会进入未分组。
4. 点击「选择本机 Excel / Choose Excel from computer」。
5. 选择客户 CSV 或 Excel 文件。
6. 等待系统显示导入结果。
7. 检查新增数量、跳过数量、更新数量和组别移动数量。
8. 使用公司名称、网站和联系人搜索，确认客户已经出现。

网页上的 Target Group 会应用到本次上传的全部客户。因此，如果要把同一份表格放入同一个组别，直接使用网页的 Target Group，不要在每一行填写不同组别。

## 6. 重复客户和更新客户

系统会按以下任一条件判断重复：

- 公司名称相同；或
- 网站的域名相同。

普通导入遇到重复客户时会跳过，不会自动覆盖。

只有确认需要更新现有客户时，才在该行的 `Force_Update` 填写 `YES`，并再次检查联系人、网站、购买角色和跟进日期。不要把整张旧客户表全部标记为 `YES`。

## 7. 上传前检查

- [ ] 文件是 `.csv`、`.xlsx` 或 `.xls`。
- [ ] 第一行是栏位名称，没有合并储存格。
- [ ] 每一行只有一个客户公司。
- [ ] 公司名称、网站和邮箱没有明显拼写错误。
- [ ] `Customer Type` 只有 `prospect` 或 `existing`。
- [ ] `Buying Role` 使用系统规定的五个值。
- [ ] `Customer Score` 只填写 `1` 至 `100` 的数字。
- [ ] 日期使用 `YYYY-MM-DD`。
- [ ] 没有把密码、API Key 或内部机密放入 CSV。
- [ ] 未确认更新时，`Force_Update` 保持空白。

## 8. 导入后验收

导入后应检查：

1. 用公司名称搜索客户。
2. 打开客户详情，确认联系人和邮箱正确。
3. 确认 Customer Type、Buying Role 和组别正确。
4. 检查 Next Follow-up Date 是否正确。
5. 不要立即重复上传同一份文件；先根据系统显示的导入结果处理错误。
